git分支开发常用命令

发表于 2016-06-30   |  

git分支开发常用命令

查看分支

git branch　//查看所有本地分支
git branch -r　//查看所有远程分支
git branch -a 　//查看所有分支
git show 分支名　//查看分支信息

创建分支

git branch 分支名

切换分支

切换分支前先提交本地变更到暂存区
git checkout 分支名

创建并切换分支

上面两步可以合并为一行
git checkout -b 分支名

注意：这里创建的是本地分支
如果多人开发，需要把本地分支提交到远程服务器

创建远程分支的本地跟踪（tracking branch）分支

git checkout -b 分支名 远程分支

同步远程分支的代码到本地

git fetch //同步所有的远程分支
git fetch 分支名 //同步单个分支的代码

提交本地的变更到远程仓库

git pull 远程仓库 分支名 //先合并远程分支的代码到本地，如果有冲突先解决冲突，pull可以理解为将fetch、merge合并为一句执行

git push 远程仓库 分支名 //提交本地的变更到远程分支

删除本地分支

删除本地分支前先提交变更到暂存区
git branch -d 分支名

如果删除失败，可使用-D强制删除
git branch -D 分支名

删除远程分支

方法一：
git push 远程仓库 --delete 分支名

删除远程不存在的分支

多人协作的时候，有人删除了一个远程分支，其他人还看得到
这是时候使用–delete删除会提示以下错误信息
error: unable to delete ‘test’: remote ref does not exist
error: failed to push some refs to ‘git@192.168.1.253:UrunNews.git’
可使用下面命令解决
git fetch -p 远程仓库

合并分支

例如现有master和develop两个分支，需要合并develop分支的代码到master

先查看当前分支是否为master分支，如果不是，使用checkout命令切换到master分支,
git checkout master
git merge develop //直接合并不生成合并记录
git merge --no-ff -m 'merge develop' develop //合并的时候生成合并记录，no fast forward

使用–no-ff

每次合并我们都应该产生一个合并记录的节点，便于版本管理，我们应该使用–no-ff来合并分支

暂存stash

场景一、你正在develop分支上开发一些新功能，这时候你的boss让你马上去修复一些线上（master分支）bug，这时候你需要checkout到master分支，并创建一个bugfix分支

场景二、你正在develop分支上开发一些新功能，这时候测试小伙伴过来告诉你上次装的build版本出了一些bug，需要你立即修复一下出个新的build版本
，这时你需要在develop上创建一个新的bugfix分支，修复build版本的bug

面对以上两种场景，需要保护工作现场，改完bug后回来接着干活，这时候stash指令就派上用场了

切换分支之前执行
git stash //保护工作现场

开发完成之后切换回develop分支接着开发，执行
git stash pop //回到工作现场

清空工作现场的变更
git stash clear

安装cocoapods

发表于 2016-04-20   |   分类于 技术   |  

1、查看当前Ruby源

gem sources -1

2、如果是默认Ruby源（被墙），删除默认源，修改为淘宝的镜像

删除默认源
gem sources --remove https://rubygems.org/

修改为淘宝源
gem sources -a https://ruby.taobao.org/

3、重复1步骤,看到下面结果表示更换ruby源成功

*** CURRENT SOURCES ***

https://ruby.taobao.org

4、安装cocospods

sudo gem install cocoapods

注意：苹果系统升级 OS X EL Capitan 后使用
sudo gem install -n /usr/local/bin cocoapods

5、设置cocoapods环境

gem setup

这步会下载repos文件，大概300多M，耗时较多，请耐心等待
我们可以使用如下命令查看下载进度

cd ~/.cocoapods
du -sh

6、更新gem

sudo gem update --system

7、新建工程，cd到工程目录

pod search 第三方库

8、创建Podfile文件

vi Podfile

9、编辑Podfile文件，可参考cocoapods官网podfile文件

inhibit_all_warnings!
platform :ios, '8.0'

target 'NewsApp' do
pod 'AFNetworking', '~>2.5.0'
end

10、安装pod

pod install --verbose --no-repo-update

SDWebImage源码学习-缓存清理机制

发表于 2016-04-05   |   分类于 技术   |  

首先安利一波SDWebImage，SDWebImage是一个异步加载图片的框架，以UIImageView分类的方式实现，深受开发者喜欢，github上排行第一的图片框架，如果你还没用过，赶紧试试看吧~

使用之前不妨先了解下SDImageCache缓存清理策略
现在打开SDImageCache.h头文件，从类名可以看出缓存是在这个类处理的

接口声明上方有这么段注释

/**
 * SDImageCache maintains a memory cache and an optional disk cache. Disk cache write operations are performed
 * asynchronous so it doesn’t add unnecessary latency to the UI.
 */

通过注释我们知道SDImageCache包含内存缓存和磁盘缓存，写磁盘缓存的操作是异步执行的，不会阻塞UI

好的，接着往下看

通过maxMemoryCost可以设置图片占用内存的最大像素值，默认为0

通过maxCacheAge可以设置图片最大缓存时间，单位为秒

打开SDWebImage.m 可以看到如下代码

static const NSInteger kDefaultCacheMaxCacheAge = 60 * 60 * 24 * 7; // 1 week

图片默认的缓存时间是一周

通过maxCacheSize可以设置磁盘缓存的最大值，单位为位，默认为0

若不设置maxCacheSize，图片会以maxCacheAge删除缓存，maxCacheAge内有较多图片缓存在磁盘的话，会占用较多的空间，建议设置一个最大值，我们可以通过下面代码设置

[[SDImageCache sharedImageCache] setMaxCacheSize:100*1024*1024]; //设置图片占用磁盘空间最大为100M
继续查看SDWebImageweb.m文件，在- (id)initWithNamespace:(NSString *)ns方法中，我们发现三个通知

这三个通知所做的工作分别是：

  • 收到内存警告的时候删除内存缓存
  • 应用程序将要终止的时候清除磁盘缓存
  • 应用程序退到后台的时候清除磁盘缓存

总结
1、图片缓存在磁盘的时间默认为1周，通过设置maxCacheAge的值，我们可以按照自己的需求设置图片在磁盘缓存的最长时间
2、通过maxMemoryCost，可设置图片占用的最大内存，当收到内存警告的通知的时候，SD会清空图片占用的所有内存
3、通过maxCacheSize，可设置图片占用的最大磁盘空间，当程序退到后台或者将要终止的时候，SD会先清理超过最大缓存时间的图片，然后计算缓存图片占用的磁盘空间，若占用的空间大于maxCacheSize，SD会将缓存的url按修改时间倒序，循环删除最旧的图片，直到，磁盘空间小于maxCacheSize/2为止

通过上面的方法，可保证图片在内存和磁盘占用的空间始终在可控的范围内

记Hexo+github搭建个人博客的过程

发表于 2015-11-13   |   分类于 技术   |  

搭建博客前的准备工作

1、安装Homebrew

Homebrew是Mac OSX上的软件包管理工具，能在Mac中方便的安装软件或者卸载软件，在终端执行下面命令安装：

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装完成后可以使用下面命令查看版本信息

npm -v

2、使用Homebrew安装node

在终端执行下面命令安装

brew install node

安装完成后可以使用下面命令查看版本信息

node -v

开始搭建个人博客

安装Hexo

npm install --verbose -g hexo-cli

初始化Hexo
进入到相应的文件夹目

hexo init

npm install

安装完成后可在目录下看到对于目录结构

.
├── _config.yml  #站点配置文件
├── package.json  #应用程序的信息
├── scaffolds  #模板文件夹，博文是根据模板生成的
├── source  #资源文件夹
|   ├── _posts  #生成的博客
└── themes  #下载的主题

现在我们执行下列命令来发布的本地服务器

hexo g #生成博客静态文件
hexo s #启动本地服务器

在浏览器中打开http://localhost:4000查看默认效果

连接github

建立Repository，建立与你用户名对应的仓库，仓库名必须为【your_user_name.github.io】，固定写法

配置_config.yml与github建立关联

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: git@github.com:FMYang/FMYang.github.io.git
  branch: master

执行配置命令

hexo d

大功告成！

快去使用http://username.github.io/访问你的个人博客试试吧。
