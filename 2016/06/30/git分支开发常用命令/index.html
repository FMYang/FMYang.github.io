git分支开发常用命令

git分支开发常用命令

查看分支

git branch　//查看所有本地分支
git branch -r　//查看所有远程分支
git branch -a 　//查看所有分支
git show 分支名　//查看分支信息

创建分支

git branch 分支名

切换分支

切换分支前先提交本地变更到暂存区
git checkout 分支名

创建并切换分支

上面两步可以合并为一行
git checkout -b 分支名

注意：这里创建的是本地分支
如果多人开发，需要把本地分支提交到远程服务器

创建远程分支的本地跟踪（tracking branch）分支

git checkout -b 分支名 远程分支

同步远程分支的代码到本地

git fetch //同步所有的远程分支
git fetch 分支名 //同步单个分支的代码

提交本地的变更到远程仓库

git pull 远程仓库 分支名 //先合并远程分支的代码到本地，如果有冲突先解决冲突，pull可以理解为将fetch、merge合并为一句执行

git push 远程仓库 分支名 //提交本地的变更到远程分支

删除本地分支

删除本地分支前先提交变更到暂存区
git branch -d 分支名

如果删除失败，可使用-D强制删除
git branch -D 分支名

删除远程分支

方法一：
git push 远程仓库 --delete 分支名

删除远程不存在的分支

多人协作的时候，有人删除了一个远程分支，其他人还看得到
这是时候使用–delete删除会提示以下错误信息
error: unable to delete ‘test’: remote ref does not exist
error: failed to push some refs to ‘git@192.168.1.253:UrunNews.git’
可使用下面命令解决
git fetch -p 远程仓库

合并分支

例如现有master和develop两个分支，需要合并develop分支的代码到master

先查看当前分支是否为master分支，如果不是，使用checkout命令切换到master分支,
git checkout master
git merge develop //直接合并不生成合并记录
git merge --no-ff -m 'merge develop' develop //合并的时候生成合并记录，no fast forward

使用–no-ff

每次合并我们都应该产生一个合并记录的节点，便于版本管理，我们应该使用–no-ff来合并分支

暂存stash

场景一、你正在develop分支上开发一些新功能，这时候你的boss让你马上去修复一些线上（master分支）bug，这时候你需要checkout到master分支，并创建一个bugfix分支

场景二、你正在develop分支上开发一些新功能，这时候测试小伙伴过来告诉你上次装的build版本出了一些bug，需要你立即修复一下出个新的build版本
，这时你需要在develop上创建一个新的bugfix分支，修复build版本的bug

面对以上两种场景，需要保护工作现场，改完bug后回来接着干活，这时候stash指令就派上用场了

切换分支之前执行
git stash //保护工作现场

开发完成之后切换回develop分支接着开发，执行
git stash pop //回到工作现场

清空工作现场的变更
git stash clear
