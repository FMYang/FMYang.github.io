SDWebImage源码学习-缓存清理机制

发表于 2016-04-05   |   分类于 技术   |  

前段时间群里有人问sd缓存的图片需要手动清除？

这类问题反映了一个普遍现象：我们在项目开发的时候，使用一些第三方框架，往往只是会使用，却很少去看源码（知其然，不知其所以然）。当然，导致这种现象的原因很多：项目开发时间紧、看不懂、懒等等因素

但想要提高技术水平，阅读优秀的源码其实是很有必要也很高效的

好的，扯了一堆废话，现在打开SD源码，看看它是如何清理缓存的

我们打开SDImageCache.h头文件，从类名可以看出缓存是在这个类处理的

接口声明上方有这么段注释
/**

* SDImageCache maintains a memory cache and an optional disk cache. Disk cache write operations are performed
* asynchronous so it doesn’t add unnecessary latency to the UI.
*/

通过注释我们知道SDImageCache包含内存缓存和磁盘缓存，写磁盘缓存的操作是异步执行的，不会阻塞UI

好的，接着往下看

/**
 * The maximum "total cost" of the in-memory image cache. The cost function is the number of pixels held in memory.
 */
@property (assign, nonatomic) NSUInteger maxMemoryCost;

/**
 * The maximum length of time to keep an image in the cache, in seconds
 */
@property (assign, nonatomic) NSInteger maxCacheAge;

/**
 * The maximum size of the cache, in bytes.
 */
@property (assign, nonatomic) NSUInteger maxCacheSize;

通过maxMemoryCost可以设置图片占用内存的最大像素值，默认为0

通过maxCacheAge可以设置图片最大缓存时间，单位为秒

打开SDWebImage.m 可以看到如下代码

static const NSInteger kDefaultCacheMaxCacheAge = 60 * 60 * 24 * 7; // 1 week

图片默认的缓存时间是一周

通过maxCacheSize可以设置磁盘缓存的最大值，单位为位，默认为0

若不设置maxCacheSize，图片会以maxCacheAge删除缓存，maxCacheAge内有较多图片缓存在磁盘的话，会占用较多的空间，建议设置一个最大值，我们可以通过下面代码设置

[[SDImageCache sharedImageCache] setMaxCacheSize:100*1024*1024]; //设置图片占用磁盘空间最大为100M
继续查看SDWebImageweb.m文件，在- (id)initWithNamespace:(NSString *)ns方法中，我们发现三个通知

[[NSNotificationCenter defaultCenter] addObserver:self
                                                 selector:@selector(clearMemory)
                                                     name:UIApplicationDidReceiveMemoryWarningNotification
                                                   object:nil];

        [[NSNotificationCenter defaultCenter] addObserver:self
                                                 selector:@selector(cleanDisk)
                                                     name:UIApplicationWillTerminateNotification
                                                   object:nil];

        [[NSNotificationCenter defaultCenter] addObserver:self
                                                 selector:@selector(backgroundCleanDisk)
                                                     name:UIApplicationDidEnterBackgroundNotification
                                                   object:nil];

这三个通知所做的工作分别是：

  • 收到内存警告的时候删除内存缓存
  • 应用程序将要终止的时候清除磁盘缓存
  • 应用程序退到后台的时候清除磁盘缓存

接着查看这三个方法具体做了什么

app收到内存警告的时候会执行下面方法
- (void)clearMemory {
    [self.memCache removeAllObjects];
}

memCache是NSCache的实例对象，NSCache是专门用来处理内存的类，使用方式和NSMutableDictionary类似

通过代码可以看到，app收到内存警告的时候SD会清空所有内存缓存的图片

应用程序将要终止的时候执行下面方法，核心代码看中文注释部分
- (void)cleanDisk {
    [self cleanDiskWithCompletionBlock:nil];
}

- (void)cleanDiskWithCompletionBlock:(SDWebImageNoParamsBlock)completionBlock {
    dispatch_async(self.ioQueue, ^{
        NSURL *diskCacheURL = [NSURL fileURLWithPath:self.diskCachePath isDirectory:YES];
        NSArray *resourceKeys = @[NSURLIsDirectoryKey, NSURLContentModificationDateKey, NSURLTotalFileAllocatedSizeKey];

        // This enumerator prefetches useful properties for our cache files.
        NSDirectoryEnumerator *fileEnumerator = [_fileManager enumeratorAtURL:diskCacheURL
                                                   includingPropertiesForKeys:resourceKeys
                                                                      options:NSDirectoryEnumerationSkipsHiddenFiles
                                                                 errorHandler:NULL];

        //获取图片过期时间
        NSDate *expirationDate = [NSDate dateWithTimeIntervalSinceNow:-self.maxCacheAge];
        NSMutableDictionary *cacheFiles = [NSMutableDictionary dictionary];
        NSUInteger currentCacheSize = 0;

        // Enumerate all of the files in the cache directory.  This loop has two purposes:
        //
        //  1. Removing files that are older than the expiration date.
        //  2. Storing file attributes for the size-based cleanup pass.
        NSMutableArray *urlsToDelete = [[NSMutableArray alloc] init];
        for (NSURL *fileURL in fileEnumerator) {
            NSDictionary *resourceValues = [fileURL resourceValuesForKeys:resourceKeys error:NULL];

            // Skip directories.
            if ([resourceValues[NSURLIsDirectoryKey] boolValue]) {
                continue;
            }

            // Remove files that are older than the expiration date;
            //若图片的修改时间超过过期时间，将图片url放到urlsToDelete数组中，稍后删除
            NSDate *modificationDate = resourceValues[NSURLContentModificationDateKey];
            if ([[modificationDate laterDate:expirationDate] isEqualToDate:expirationDate]) {
                [urlsToDelete addObject:fileURL];
                continue;
            }
            //计算剩余图片所占用的空间
            // Store a reference to this file and account for its total size.
            NSNumber *totalAllocatedSize = resourceValues[NSURLTotalFileAllocatedSizeKey];
            currentCacheSize += [totalAllocatedSize unsignedIntegerValue];
            [cacheFiles setObject:resourceValues forKey:fileURL];
        }

        //删除过期的图片
        for (NSURL *fileURL in urlsToDelete) {
            [_fileManager removeItemAtURL:fileURL error:nil];
        }

        // If our remaining disk cache exceeds a configured maximum size, perform a second
        // size-based cleanup pass.  We delete the oldest files first.
        //最大缓存值大于0，当前缓存值大于最大占用的缓存值，按照缓存时间，删除最旧的一半的图片
        if (self.maxCacheSize > 0 && currentCacheSize > self.maxCacheSize) {
            // Target half of our maximum cache size for this cleanup pass.
            const NSUInteger desiredCacheSize = self.maxCacheSize / 2;

            // Sort the remaining cache files by their last modification time (oldest first).
            NSArray *sortedFiles = [cacheFiles keysSortedByValueWithOptions:NSSortConcurrent
                                                            usingComparator:^NSComparisonResult(id obj1, id obj2) {
                                                                return [obj1[NSURLContentModificationDateKey] compare:obj2[NSURLContentModificationDateKey]];
                                                            }];

            // Delete files until we fall below our desired cache size.
            for (NSURL *fileURL in sortedFiles) {
                if ([_fileManager removeItemAtURL:fileURL error:nil]) {
                    NSDictionary *resourceValues = cacheFiles[fileURL];
                    NSNumber *totalAllocatedSize = resourceValues[NSURLTotalFileAllocatedSizeKey];
                    currentCacheSize -= [totalAllocatedSize unsignedIntegerValue];

                    if (currentCacheSize < desiredCacheSize) {
                        break;
                    }
                }
            }
        }
        if (completionBlock) {
            dispatch_async(dispatch_get_main_queue(), ^{
                completionBlock();
            });
        }
    });
}
应用程序退到后台的时候执行下面方法
- (void)backgroundCleanDisk {
    UIApplication *application = [UIApplication sharedApplication];
    __block UIBackgroundTaskIdentifier bgTask = [application beginBackgroundTaskWithExpirationHandler:^{
        // Clean up any unfinished task business by marking where you
        // stopped or ending the task outright.
        [application endBackgroundTask:bgTask];
        bgTask = UIBackgroundTaskInvalid;
    }];

    // Start the long-running task and return immediately.
    [self cleanDiskWithCompletionBlock:^{
        [application endBackgroundTask:bgTask];
        bgTask = UIBackgroundTaskInvalid;
    }];
}

删除图片的逻辑与上面一样，不同的是在后台task执行

通过源码分析，我们知道SD是如何清理缓存的了

总结
1、图片缓存在磁盘的时间默认为1周，通过设置maxCacheAge的值，我们可以按照自己的需求设置图片在磁盘缓存的最长时间
2、通过maxMemoryCost，可设置图片占用的最大内存，当收到内存警告的通知的时候，SD会清空图片占用的所有内存
3、通过maxCacheSize，可设置图片占用的最大磁盘空间，当程序退到后台或者将要终止的时候，SD会先清理超过最大缓存时间的图片，然后计算缓存图片占用的磁盘空间，若占用的空间大于maxCacheSize，SD会将缓存的url按修改时间倒序，循环删除最旧的图片，直到，磁盘空间小于maxCacheSize/2为止

通过上面的方法，可保证图片在内存和磁盘占用的空间始终在可控的范围内
